# Field Mapping Research

Map the terrain before committing to a thesis.

Read shared references:
- `../researcher/references/core-method.md` for `FRAME` and `MAP`
- `../researcher/references/source-hierarchy.md` for source-class coverage
- `../researcher/references/query-shaping.md` for first-pass query design
- `../researcher/references/frontier-management.md` for frontier construction

## Workflow

1. Frame the scope, unknowns, and relevant source classes.
2. Build a broad evidence map across official, behavioral, human, operator, and
   adversarial signals.
3. Name the major branches, actors, debates, and blind spots.
4. Convert the map into 2-4 frontier leads with clear next actions.
5. Stop after the field map and frontier unless the user explicitly wants deep
   validation.

## Constraints

- Do not over-deepen during the mapping pass.
- Do not confuse many URLs with good coverage.
- Include at least one contradiction or skepticism path in the map.

## Output

Create or update `research-{topic}.md` while working so the map and frontier
are visible.

Prefer sections like:
- Problem Frame
- Field Map
- Key Actors And Branches
- Source Landscape
- Active Frontier
- Open Questions
