# Source Pack: Field Signals And Lived Experience

Load this file when the task needs real practitioner sentiment, day-to-day
friction, or community reality.

## 1. Best places to look

Use:
- Reddit and specialist forums for detailed pain points
- X for fast-moving narratives, disagreements, and operator commentary
- LinkedIn for role descriptions, trajectories, and public professional norms
- GitHub issues and repos for real technical friction
- Blind and Glassdoor for employer-specific internal sentiment

## 2. How to use field signals

Treat these as:
- signal sources
- friction sources
- challenge sources
- language and vocabulary sources

Do not treat a single post as truth. Look for repeated patterns across people,
platforms, and time.

## 3. What to extract

Focus on:
- repeated complaints
- repeated hidden requirements
- day-to-day task reality
- "what outsiders get wrong"
- disagreement between beginners and experienced operators
