# Core Method

Load this file when you need the shared research workflow.

## 1. FRAME

Define the task before searching:
- decision question or judgment target
- key claims that may need confirmation or falsification
- important stakeholders and their incentives
- source classes likely to matter
- what would change the answer

## 2. MAP

Run a controlled first pass. Cover evidence classes, not just topic branches.

Aim to surface:
- official narrative
- behavioral reality
- operator or expert view
- lived experience
- adversarial or contradiction path

Stop the first pass after roughly 4-6 search actions and write a field map.

## 3. FRONTIER

Convert the map into 2-4 active leads. Each lead should include:
1. what the lead is
2. why it matters
3. which source class it touches
4. what the next action is
5. what claim it could confirm or weaken

## 4. DEEPEN

For each active branch, run:

```text
FETCH / READ → EXTRACT → CHALLENGE → NEXT JUMP
```

After each step, log:
- finding
- source
- contradiction or caveat
- gap
- next jump

Do not keep widening with generic searches once lead-driven work has started.

## 5. CHALLENGE

For every strong thesis, actively look for:
- direct rebuttal
- failed example
- conflicting user experience
- hidden incentive
- evidence that the narrative is ahead of the reality

## 6. SYNTHESIZE

Separate the answer into:
- confirmed
- likely
- contested
- unresolved
- what could change the view

Mark source quality:
- `✓` primary, artifact, or directly read high-value source
- `?` secondary, incomplete, or weakly supported
- `✗` unresolved, contradictory, or source problem
